# AGENTS.md

## Обзор проекта
«Заботливый Педиатр» — интерактивный веб-сервис для родителей: персональный сайт врача-педиатра с умным ИИ-ассистентом (на базе Gemini API), картой ребенка, календарем прививок, калькулятором развития (ростовые графики) и квиз-игрой о детском здоровье.
Проект состоит из клиентской части на React (SPA) и серверной части на Express.

## Команды верификации
Все команды должны выполняться из корня проекта:
- Установка зависимостей: `npm install`
- Запуск в режиме разработки: `npm run dev` (запускает `tsx server.ts` на порту из env, по умолчанию `PORT=3000`)
- Сборка проекта (frontend + backend): `npm run build` (собирает React-приложение во временную папку `dist` и бандлит `server.ts` в `dist/server.js` с помощью `esbuild`)
- Запуск продакшн-сервера: `npm start` (запускает `node dist/server.js`)
- Проверка линтера: `npm run lint` (выполняет `tsc --noEmit`)
- Запуск тестов: `npm test` (запускает тесты через `node --test` для `src/**/*.test.ts`)
- Очистка собранных файлов: `npm run clean`

*Все указанные выше команды были проверены в ходе текущего аудита (июль 2026) и завершились успешно.*

## Архитектурные ограничения
- **Стек технологий**: React 19, TypeScript 5.8, Express 4.21, Vite 6.2, TailwindCSS 4.
- **Главные файлы логики**:
  - [src/App.tsx](file:///m:/Projects/AI%20STUDIO/%D1%81%D0%B0%D0%B9%D1%82%20%D0%BF%D0%B5%D0%B4%D0%B8%D0%B0%D1%82%D0%BE%D1%80/src/App.tsx) — монолитный файл фронтенда (содержит роутинг, интерфейс чата, калькуляторы, профили, квизы).
  - [src/clinicalRouter.ts](file:///m:/Projects/AI%20STUDIO/%D1%81%D0%B0%D0%B9%D1%82%20%D0%BF%D0%B5%D0%B4%D0%B8%D0%B0%D1%82%D0%BE%D1%80/src/clinicalRouter.ts) — детерминированный клинический маршрутизатор симптомов и красных флагов.
  - [server.ts](file:///m:/Projects/AI%20STUDIO/%D1%81%D0%B0%D0%B9%D1%82%20%D0%BF%D0%B5%D0%B4%D0%B8%D0%B0%D1%82%D0%BE%D1%80/server.ts) — сервер Express (API, демо-авторизация, GCM шифрование профилей, аудит-логирование, раздача статики).
- **Безопасность**:
  - Любое изменение API эндпоинтов должно сопровождаться проверкой наличия заголовков безопасности, проверкой авторизации сессии (Bearer token), проверкой согласия на обработку данных (`consentAccepted`).
  - Данные профилей шифруются алгоритмом `aes-256-gcm` перед сохранением в store.
  - Запрещено передавать в Gemini API невалидированные, слишком длинные сообщения или сообщения, содержащие красные клинические флаги (они перехватываются сервером).
- **Производительность**:
  - Тяжелые библиотеки (например, `jspdf`, `recharts` / `GrowthChart`) должны импортироваться динамически / лениво, чтобы размер главного бандла фронтенда не превышал 500 kB.
